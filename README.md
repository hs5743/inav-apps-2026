# 人際導航員

人際導航員是一套給國中小 SEL、人際互動與班級經營課程使用的即時協作網站。學生以小組形式進入同一個房間，閱讀校園人際情境，從策略卡中選擇合適做法，輪流說明理由，最後投票找出較安全、尊重、有效的回應方式。

網站目前部署方向為 GitHub Pages 靜態前端搭配 Firebase Cloud Firestore 即時資料庫，學生端免登入，教師後台以 Google 登入保護。

## 網站主要功能概要

- **小組即時遊戲房間**：組長建立房間，組員用同一組代碼加入。
- **SEL 情境任務**：每輪隨機抽出校園、人際或網路互動情境。
- **策略卡選擇**：每位玩家取得 5 張隨機策略卡與 1 張空白絕招卡。
- **限時節奏控制**：出牌倒數 90 秒、投票倒數 60 秒，組長可暫停討論。
- **同儕發表與投票**：玩家出牌後說明理由，再投票選出較合適的做法。
- **人數與體驗控制**：每房最多 8 人，建議 4-6 人一組。
- **組長控制台**：開始任務、進入投票、公布結果、廣播提醒、結束遊戲。
- **教師後台**：查看房間、活動摘要、策略類型統計，匯出 CSV/HTML 報告。
- **隱私友善**：建議使用座號、暱稱或組別，不要求學生帳號登入。

## 網站發展歷程概要

### 1. Apps Script + Google Sheets 原型

早期版本以 Google Apps Script 搭配 Google Sheets 作為簡易後端資料庫。這個架構容易上手，但在多人同步時可能出現延遲與停頓，也不利於大規模展示或即時互動。

### 2. Firebase 即時協作版

後續改為靜態網站搭配 Firebase Cloud Firestore，改善多人同步體驗。房間、玩家、出牌、投票與階段狀態都改由 Firestore 管理，適合課堂與研討會現場使用。

### 3. 研討會模式與用量優化

為了支援約百人規模的教育研討會展示，網站加入小組人數上限、倒數節奏、組長暫停控制，並調整同步策略：一般玩家主要監聽自己的狀態與房間節奏，組長掌握全組進度，以降低資料庫 reads 放大。

### 4. 視覺與角色升級

首頁加入奇幻導航感主視覺，整站調整為更有活力的藍綠、珊瑚、金黃、紫色配色系統。玩家頭像改為可愛角色圖，讓遊戲更有辨識度與沉浸感。

### 5. 教師後台與活動摘要

新版教師後台加入活動後摘要：平均每房輪數、策略類型排行、單一房間逐輪摘要，以及 CSV/HTML 匯出，方便教師事後教學研討與紀錄整理。

## 網站主要功能介紹

### 首頁與進入房間

首頁提供學生加入房間與組長建立房間的入口。使用者輸入姓名、座號或暱稱，再輸入房間代碼即可加入。首頁也提供 8 種導航頭像供玩家選擇。

建議課堂使用時採用：

- 房間代碼：桌號、組別或班級代碼，例如 `A1`、`B2`、`G1`
- 玩家名稱：座號、暱稱或組別代號
- 小組人數：4-6 人最佳，最多 8 人

### 情境任務

每輪開始時，系統會隨機抽出一個情境。情境主題涵蓋同儕衝突、合作分組、網路互動、情緒界線與安全判斷等 SEL 常見議題。

玩家需要先閱讀情境，思考其中可能出現的感受，再選擇合適的策略卡。

### 策略卡

每位玩家每輪會取得 6 張卡：

- 5 張隨機策略卡
- 1 張空白絕招卡

策略卡包含策略名稱、說明、培養能力與適用時機。空白絕招可讓玩家提出牌庫之外的做法，並說明為什麼安全、尊重、有效。

若玩家資料異常導致手牌為空，系統會自動補發策略卡，避免活動中斷。

### 組長控制台

組長可在遊戲中控制流程：

- 開始新任務
- 進入發表投票
- 公布本輪結果
- 暫停倒數討論
- 繼續倒數
- 發送組長廣播提醒
- 結束遊戲

結束遊戲前會出現確認提示，避免誤觸。

### 倒數與討論節奏

目前預設：

- 出牌階段：90 秒
- 投票階段：60 秒

倒數不是強制處罰，而是讓玩家知道還有多久，減少等待焦慮。組長可依現場狀況暫停倒數，作為小組討論或教師引導時間。

### 發表與投票

玩家出牌後，組長可安排成員輪流說明選擇理由。進入投票階段後，玩家聽完同學說明，再選出較符合「安全、尊重、有效」原則的策略。

此設計的重點不是競爭，而是讓學生練習比較不同人際做法的優缺點。

### 本輪結果

組長公布結果後，網站會顯示本輪得票狀況。此時可引導學生討論：

- 哪個策略最安全？
- 哪個策略最尊重自己與他人？
- 哪個策略最可能有效改善情境？
- 是否還需要找老師、家長或可信任的大人協助？

### 教師後台

教師後台使用 Firebase Authentication 的 Google 登入保護。授權教師登入後可查看房間列表與活動資料。

教師後台目前支援：

- 查看房間代碼、階段、輪次、人數、更新時間
- 查看全站活動摘要
- 查看平均每房完成輪數
- 查看常用策略類型統計
- 查看單一房間逐輪摘要
- 匯出 CSV 報告
- 匯出 HTML 報告
- 解散房間並刪除玩家資料

活動摘要採教學研討用途設計，保存策略類型、情境、出牌數、投票數與最高票策略等資訊，不保存完整個人投票流向。

## 教育價值

人際導航員的核心價值，是將 SEL 從抽象口號轉化為具體情境中的選擇練習。

學生在遊戲中會練習：

- 讀懂情境與感受
- 選擇合適人際策略
- 用理由支持自己的選擇
- 聆聽不同觀點
- 比較做法是否安全、尊重、有效
- 在小組中形成共識

教師則可透過後台摘要觀察：

- 學生偏好哪些策略類型
- 哪些情境最容易引發討論
- 小組是否能完成多輪互動
- 高票策略是否符合課程目標

## 建議使用方式

### 15 分鐘快速體驗

1. 教師說明「安全、尊重、有效」三項判準。
2. 學生分組並加入房間。
3. 組長開始第一輪任務。
4. 玩家選牌、說明理由並投票。
5. 公布結果，進行簡短討論。

### 30 分鐘完整活動

1. 5 分鐘：說明活動與分組。
2. 8 分鐘：第一輪任務。
3. 8 分鐘：第二輪任務。
4. 5 分鐘：小組回顧。
5. 4 分鐘：全班分享高票策略或教師總結。

### 研討會展示

- 每組 4-6 人。
- 房間代碼使用桌號。
- 玩 1-2 輪即可展示核心流程。
- 最後進入教師後台，展示活動摘要與匯出報告。

## 技術架構

- 前端：HTML、CSS、JavaScript 靜態網站
- 部署：GitHub Pages
- 資料庫：Firebase Cloud Firestore
- 教師登入：Firebase Authentication + Google 登入
- 資料集合：
  - `rooms`
  - `rooms/{roomId}/players`

## 本機預覽

可用任一靜態伺服器開啟，例如：

```powershell
npx.cmd serve .
```

或直接使用 VS Code / 瀏覽器開啟 `index.html`。正式分享建議使用 GitHub Pages。

## 安全與隱私提醒

目前設計重點是課堂與研討會的便利使用，因此學生端不要求登入。建議教師提醒學生使用座號、暱稱或組別，不輸入完整真名。

教師後台使用 Google 登入與 Firestore 規則保護，不使用前端公開密碼。若要長期大規模正式營運，建議後續再加入 App Check、更細緻的房間權限或後端驗證機制。
